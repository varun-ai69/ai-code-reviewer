# Contributing to RepoSage

First off, thank you for considering contributing to **RepoSage**! It's people like you that make the open-source community such a fantastic place to learn, inspire, and create.

## 🚀 How to Contribute

1. **Fork the repository** to your own GitHub account.
2. **Clone the project** to your local machine.
3. **Create a branch** for your feature or bug fix:
   `git checkout -b feature/your-feature-name` or `git checkout -b bugfix/issue-number`
4. **Make your changes** and commit them with clear, descriptive messages.
5. **Push to your fork** and submit a **Pull Request** to our `main` branch.

## 🏷️ Issue Labels

If you are participating in **GSSoC** or just looking for a place to start, look for issues with the following labels:
- `good first issue` - Perfect for beginners.
- `beginner friendly` - Easy tasks with mentorship available.
- `frontend` / `backend` / `ai` - Specific module tasks.
- `gssoc` - Officially tracked issues for the event.

## 🛠️ Code Style

- **Frontend**: Follow standard React best practices and use ESLint.
- **Backend**: Use ES6 syntax, clear async/await structures, and document APIs.
- **AI Engine**: Follow PEP8 standards for Python and use type hints where applicable.

Please respect our [Code of Conduct](CODE_OF_CONDUCT.md) in all interactions.
